GRAVITY FLIP — COMPLETE PROJECT OVERVIEW

---

1. CORE CONCEPT

Gravity Flip is a high-speed 2D endless runner where the player controls a character moving through a narrow corridor while avoiding obstacles by flipping gravity.

The player does not control movement speed or direction—only gravity.

Core mechanic:

- Tap → instantly invert gravity (floor ↔ ceiling)

Core goal:

- Survive as long as possible
- Achieve the highest score (distance-based)

---

2. CORE GAME LOGIC

2.1 Player Movement Logic

- Character runs automatically forward at constant speed
- Gravity pulls the character toward:
  - Floor (default)
  - Ceiling (after flip)
- Flip is instant but includes a very short transition animation (0.1–0.2s)

2.2 Collision System

- Collision detection uses simple bounding box or capsule collider
- Collision with any obstacle results in death
- Optional grace mechanic: 1–2 pixel forgiveness window for fairness

2.3 Score System

- Score increases continuously based on time survived

Formula:

- Score += 1 per second
- Bonus:
  - Near-miss: +2
  - Perfect flip: +5

2.4 Speed Scaling

- Speed increases gradually over time

Example:

- 0–20 sec: slow
- 20–40 sec: medium
- 40–60 sec: fast
- 60+ sec: extreme

Speed multiplier increases every 10 seconds

3. MAP / LEVEL DESIGN

3.1 Map Structure

- Infinite procedurally generated corridor
- Two parallel surfaces:
  - Floor
  - Ceiling

3.2 Map Segments

The map is built from reusable segments:

- Straight path
- Spike patterns
- Gap sequences
- Moving obstacle zones
- Tight corridor zones

Each segment:

- Length: 2–5 seconds of gameplay
- Randomized but weighted by difficulty

3.3 Biomes / Environments

Neon Core (Default)

- Dark background
- Cyan/purple glowing obstacles

Cyber Grid

- Grid background
- Electric blue lines

Lava Tunnel

- Red/orange glowing cracks
- Heat distortion

Ice Cavern

- Blue/white tones
- Subtle fog effect

Each biome rotates every 30–60 seconds

---

4. OBSTACLES

4.1 Basic Obstacles

Floor Spikes

- Triangular shapes
- Fixed to floor

Ceiling Spikes

- Inverted triangles

Spike Walls

- Full vertical barriers with small gaps

---

4.2 Advanced Obstacles

Moving Spikes

- Move between floor and ceiling
- Predictable timing

Rotating Blades

- Circular with spinning motion
- Radius-based hitbox

Laser Gates

- Horizontal beams
- Activate/deactivate in intervals

Crushing Blocks

- Move inward from floor and ceiling
- Timed traps

Narrow Tunnels

- Very tight gaps requiring perfect timing

---

5. LEVEL / DIFFICULTY SYSTEM

No traditional levels — uses dynamic difficulty scaling

Difficulty Layers

Phase 1 (0–15s)

- Basic spikes
- Wide gaps

Phase 2 (15–30s)

- Alternating spikes
- Faster pacing

Phase 3 (30–50s)

- Moving obstacles
- Laser introduction

Phase 4 (50–70s)

- Tight spaces
- Mixed patterns

Phase 5 (70s+)

- Extreme speed
- Complex combinations
